## What to delete from the template

- `Components/Pages/Counter.razor` — template demo, not needed
- `Components/Pages/Weather.razor` — template demo, not needed
- `Components/Pages/Home.razor` — replace with a redirect to `/new-entry`, or delete

## Key decisions

- `Models/` holds both EF-generated classes and form/display models in one folder — no point splitting into subfolders for a 2-page app.
- `Services/` has just one file — no subfolder needed.
- `wwwroot/js/` isolates JS from CSS cleanly without adding extra nesting.
