## What lives where and why

- **Components/Layout/** — the shell every page sits inside. Change the nav or page wrapper here.
- **Components/Pages/** — the two actual screens (New Entry form + Records table). Most feature work happens here.
- **Models/** — all C# data shapes in one flat folder. EF-generated files sit alongside hand-written ones; no sub-folders needed for a 2-page app.
- **Services/** — one file, one job: read from and write to the database.
- **wwwroot/js/** — the only JS in the project. It lives here because reading the user's timezone can only be done in the browser, not on the server.
- **sql/** — scratch queries for checking data during development. Safe to ignore otherwise.
- **setup.sql** — run this once before launching the app for the first time to create the DB schema.
